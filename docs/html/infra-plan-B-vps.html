技術詳細・セットアップ手順
最安構成

VPS 上のソフトウェア構成

Nginx

リバースプロキシ。Let's Encrypt SSL 証明書自動更新。静的ファイル配信。Gzip 圧縮。レート制限。

Django + Gunicorn

WSGI サーバー。Systemd でプロセス管理。ワーカー数 2〜4。Unix ソケット接続で高速化。

PostgreSQL 16

ローカルインストール。自動バックアップ（pg_dump cron）。パフォーマンスチューニング可能。

Redis 7

ローカルインストール。Celery ブローカー。メモリ制限設定推奨。永続化設定可能。

セットアップ手順（概要）

  1. VPS 作成: Vultr で Ubuntu 22.04 LTS インスタンスを作成（Tokyo リージョン）
  2. 初期設定: SSH 鍵設定、ファイアウォール（ufw）、fail2ban、非 root ユーザー作成
  3. ミドルウェア: PostgreSQL 16、Redis 7、Nginx、Python 3.12 をインストール
  4. アプリ設定: Django プロジェクト配置、Gunicorn + Systemd ユニットファイル作成
  5. Celery 設定: Celery Worker の Systemd ユニットファイル作成
  6. SSL: Certbot で Let's Encrypt SSL 証明書を取得・自動更新設定
  7. デプロイ: Git pull → collectstatic → migrate → Gunicorn restart のスクリプト作成
  8. バックアップ: pg_dump + cron で日次バックアップ（外部ストレージへ転送推奨）

メリット・デメリット

メリット

  • 最安コスト（$6〜12/月）
  • 完全なサーバーコントロール
  • 外部サービス依存なし
  • レイテンシ最小（全て同一マシン）
  • 無料枠の制限に縛られない
  • カスタムチューニング自由

デメリット

  • サーバー管理スキルが必須
  • OS・ミドルウェアの更新が必要
  • セキュリティパッチ適用が自己責任
  • 単一障害点（1台に全集約）
  • スケールアウトが困難
  • バックアップ設計が必要
  • デプロイの自動化が別途必要
  • 障害時は自力で復旧
注意: VPS 構成では production.py の Neon/Upstash/R2 関連の設定を、ローカルの PostgreSQL/Redis/ファイルシステムに変更する必要があります。デプロイスクリプトやバックアップスクリプトも 別途作成が必要です。構成Aと比較して、初期セットアップと継続的な運用に追加工数が発生します。
