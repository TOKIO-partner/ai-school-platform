技術詳細・セットアップ手順
推奨構成

技術スタック詳細

Vercel（フロントエンド）

Next.js 16 + React 19。Edge Network による高速配信。Git push で自動デプロイ。Preview URL で PR レビュー。

Render（バックエンド）

Django + Gunicorn。WhiteNoise で静的ファイル配信。Blueprint（render.yaml）で一括構築。自動 SSL/TLS。

Neon（データベース）

サーバーレス PostgreSQL。自動スケール＆コールドスタート。ブランチ機能で開発/本番分離。SSL 必須接続。

Upstash（Redis）

サーバーレス Redis。TLS 暗号化接続（rediss://）。REST API 対応。Celery ブローカー＆結果バックエンド。

Cloudflare R2（ストレージ）

S3 互換 API。egress 無料。django-storages で直接連携。カスタムドメインでメディア配信。

Cloudflare（CDN）

グローバル CDN。DDoS 保護。SSL/TLS 自動管理。DNS プロキシでオリジン IP 秘匿。

セットアップ手順（概要）

  1. Neon: プロジェクト作成 → DATABASE_URL 取得 → Django migrate 実行
  2. Upstash: Redis DB 作成（Singapore） → CELERY_BROKER_URL 取得
  3. Cloudflare R2: バケット作成 → API Token 発行 → 環境変数設定
  4. Render: Web Service + Background Worker 作成 → 環境変数一括設定 → デプロイ
  5. Vercel: NEXT_PUBLIC_API_URL を Render URL に設定
  6. Cloudflare DNS: api.momocri.com → Render CNAME, SSL Full(strict) 設定

メリット・デメリット

メリット

  • サーバー管理・OS更新が不要
  • Git push のみでデプロイ完了
  • 各サービスが独立してスケール可能
  • 無料枠の組み合わせでコスト最適化
  • SSL/TLS が自動で設定される
  • 障害時はサービス側が自動復旧
  • production.py が既に実装済み

デメリット

  • Render 無料枠はコールドスタートあり
  • Neon Free は 0.5GB ストレージ制限
  • Upstash Free は 10,000コマンド/日制限
  • 複数サービスの管理画面が分散
  • ベンダーロックインのリスク
  • カスタマイズの自由度に制限あり
