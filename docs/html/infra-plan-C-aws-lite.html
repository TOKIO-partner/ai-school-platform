AWS サービス詳細

Lightsail

固定料金の VPS。Nginx + Gunicorn + Celery を同居。スナップショット・バックアップ付き。EC2 へのアップグレードパスあり。

RDS PostgreSQL

マネージド DB。自動バックアップ（7日保持）。自動パッチ適用。Multi-AZ 対応（将来のスケールアップ時）。

ElastiCache Redis

マネージド Redis。高可用性。暗号化対応。Celery ブローカー + セッションストアに利用。

S3

オブジェクトストレージ。django-storages で直接連携。ライフサイクルポリシー。バージョニング対応。

CloudFront

グローバル CDN。S3 オリジン連携。SSL/TLS 自動管理（ACM 証明書無料）。キャッシュ最適化。

Route 53

DNS サービス。ヘルスチェック対応。Alias レコードで AWS サービスと統合。レイテンシベースルーティング。

セットアップ手順（概要）

  1. VPC 設定: VPC + サブネット作成、セキュリティグループ設定
  2. RDS: PostgreSQL インスタンス作成 → パラメータグループ設定 → DATABASE_URL 取得
  3. ElastiCache: Redis クラスター作成 → セキュリティグループで Lightsail からのアクセスを許可
  4. S3: バケット作成 → IAM ユーザー + ポリシー作成 → CORS 設定
  5. Lightsail: インスタンス作成 → VPC Peering 設定 → Django デプロイ
  6. CloudFront: ディストリビューション作成 → S3 オリジン設定 → カスタムドメイン設定
  7. Route 53: ホストゾーン作成 → A/CNAME レコード設定

メリット・デメリット

メリット

  • AWS エコシステムの豊富なサービス
  • RDS の自動バックアップ・パッチ
  • Lightsail → EC2 → ECS へのスムーズな移行
  • CloudWatch によるモニタリング
  • IAM による詳細なアクセス制御
  • 将来の大規模化に最も強い
  • エンタープライズ契約・サポート対応

デメリット

  • 3構成中で最もコストが高い
  • AWS の学習コスト・複雑さ
  • 設定項目が多く初期構築に時間
  • VPC/セキュリティグループの設計が必要
  • 従量課金要素ありでコスト予測が難しい
  • MVP フェーズではオーバースペック
